# 需求理解 Agent

> 索引：本项目的 `project-rules/subagents.md`（在那里登记触发词与本 playbook 的映射）

**触发词**："理解需求 / 这个需求是啥 / 帮我看下这个 PRD"

**目标**：把模糊需求落到「现状代码 + 历史决策 + 待澄清点」三段式输出。

**必做前置**：必查 memory，必扫代码现状。

## 编排

1. **记忆扫描**（按 [../memory.md](../memory.md)，并行执行）：
   - 读 `MEMORY.md` 索引 → 命中条目读对应 `.md`
   - 若配置了记忆类 MCP（向量检索 / 长期记忆等），关键词补搜
2. **代码现状**：用 `Explore` agent 找受影响模块（不要自己散弹 grep；让 Explore 出报告）
3. **PRD 解析**（如果用户给的是文档 URL / .md 文件）：若项目接了 PRD 工作流 skill，走其导入入口
4. **需求拆解**：调需求分析类 agent，把模糊点列出来

## 输出格式

```text
## 需求范围
- [一句话总结]

## 现状代码（来自 Explore）
- 受影响文件: [...]
- 相关已有抽象: [...]

## 历史相关决策（标注来源）
- [来自记忆文件 xxx.md] / [来自记忆 MCP 搜索]

## 待澄清问题（必须用户回答才能进入实现）
- [ ] ...
```

## 红线

呼应 [../soul.md](../soul.md) 第 1 条「不假设、不隐藏困惑」——有歧义必须列出来问，不要替用户选。
